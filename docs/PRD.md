# PRD：基于个人错题的学科应试咨询 Agent

## 1. 产品定位

### 1.1 用户群体
当前版本针对**有一定基础但需要提升的中等同学**（例如：满分 100 分、考试 70 分的同学）。

### 1.2 价值主张
我们提供基于个人错题集的学科专业咨询，为你把控明确的学习方向及精准的学习目标。

### 1.3 解决的核心问题
1. **明确方向**：基于个人错题解决用户不知道当前学习进展、或哪些板块是薄弱点等方向性的问题。
2. **解决方案**：具体落实到精准题目并提供通用母题思考路径，解决"对该类型题目无法下手"的问题。

---

## 2. 交互流程

1. 我（开发者 / 产品经理）配置学科的教材、考纲以及知识点的关联资料，作为**学科知识库**——暂定科目：高中数学。
2. 用户选择学习科目——暂定只能选择高中数学。
3. 用户上传个人应试 / 考试真题 / 错题本，作为**个人错题知识库**。
4. Agent 基于学科知识库的内容，具体分析个人错题知识库，给出相关诊断分析。
5. 用户可进行答题，并将错误题目上传至个人错题知识库。
6. 基于错题知识库给出分析……如此循环。

---

## 3. 用户动线

1. 用户选择学习科目
2. 用户上传并存档个人考试真题 / 错题
3. 用户阅读诊断分析
4. 用户做题
5. 用户阅读诊断分析……

---

## 4. 诊断分析报告的具体内容

> 以下板块全部需要基于两个知识库（学科知识库 + 个人错题知识库）进行推测。

### 4.1 知识图谱（TBC——MVP 只做文字关联）
需要切片并标识用户当前错误题目比较多的知识点 / 章节在当前知识图谱中的哪个板块。

**目的**：让用户 get 到"我的薄弱点"在整个知识点中的位置，以及和其他知识点之间的关系。

### 4.2 方向分析
1. 明确用户当前薄弱的章节，以及薄弱章节之间存在的关联。
2. （TBC——答题形式）基于分析用户当前错题，分析用户在思维模式或思考路径上的短板，并给建议。
3. （TBC）**学习路径**：给出用户科学、具体的学习方案
   - 本周目标
   - 2 小时内目标

### 4.3 真题模拟
基于用户当前短板，从知识库切片类似知识点的真题投放：

- **内容**：
  1. 真题题目 / 选项
  2. 标准答案
  3. 通用母题的思考路径
- **UX**：
  1. 只展示题目 / 选项，不展示答案
  2. 用户提交当前题目后，给予标准答案及通用题目的思考路径

---

## 5. MVP 版本材料

- 限定 2 个章节，以**函数和导数**为例即可，无需准备全部知识。
